# Лабораторная работа: Сравнительный анализ производительности REST API фреймворков

## Выбранные фреймворки

| Тип | Фреймворк |
|-----|-----------|
| Синхронный | Flask |
| Асинхронный | Sanic |
| Асинхронный | Quart |

## Реализация API

На каждом фреймворке реализовано два эндпоинта:

- `/cpu` – выполняет CPU-интенсивную задачу (суммирование 5 млн чисел) в блокирующем режиме
- `/cpu_fixed` – выполняет ту же задачу, но неблокирующим способом (через `asyncio.to_thread()` для асинхронных фреймворков)

## Условия тестирования

- Все серверы запущены на одной машине
- Количество воркеров: 1 для каждого фреймворка
- CPU-задача одинаковая для всех фреймворков
- Инструмент: Locust (headless режим)
- Длительность теста: 20 секунд на сценарий
- Нагрузка: 50, 100 и 200 одновременных пользователей

## Собранные метрики

- RPS (запросов в секунду)
- Средняя latency (время ответа)
- p95 (95% ответов укладываются в это время)
- Процент ошибок

## Результаты тестирования

### Flask (синхронный)

| Пользователи | Эндпоинт | RPS | Средняя latency | p95 | Ошибки |
|--------------|----------|-----|-----------------|-----|--------|
| 50 | /cpu | 1.8 | 8574 мс | 14000 мс | 0% |
| 50 | /cpu_fixed | 1.6 | 6848 мс | 13000 мс | 0% |
| 100 | /cpu | 0.9 | 14263 мс | 19000 мс | 0% |
| 100 | /cpu_fixed | 1.3 | 14170 мс | 19000 мс | 0% |

**Вывод:** Flask не справляется с CPU-интенсивными задачами. При росте нагрузки производительность падает.

### Sanic (асинхронный)

| Пользователи | Эндпоинт | RPS | Средняя latency | p95 | Ошибки |
|--------------|----------|-----|-----------------|-----|--------|
| 50 | /cpu | 81.6 | 1.6 мс | 3 мс | 0% |
| 50 | /cpu_fixed | 81.5 | 1.6 мс | 3 мс | 0% |
| 100 | /cpu | 161.0 | 2.2 мс | 4 мс | 0% |
| 100 | /cpu_fixed | 162.6 | 2.1 мс | 4 мс | 0% |
| 200 | /cpu | 323.0 | 3.4 мс | 7 мс | 0% |
| 200 | /cpu_fixed | 327.0 | 3.6 мс | 7 мс | 0% |

**Вывод:** Sanic показывает отличную производительность. RPS растет пропорционально количеству пользователей, время ответа остается низким.

### Quart (асинхронный)

| Пользователи | Эндпоинт | RPS | Средняя latency | p95 | Ошибки |
|--------------|----------|-----|-----------------|-----|--------|
| 50 | /cpu | 0.7 | 10551 мс | 16000 мс | 0% |
| 50 | /cpu_fixed | 0.6 | 10238 мс | 18000 мс | 0% |

**Вывод:** Несмотря на асинхронность, Quart показывает результаты на уровне синхронного Flask. Это связано с неправильной реализацией (отсутствует вынос в поток).

## Сравнительный анализ

| Показатель | Flask (50 users) | Sanic (50 users) | Quart (50 users) |
|------------|------------------|------------------|------------------|
| RPS | 1.8 | 81.6 | 0.7 |
| Средняя latency | 8574 мс | 1.6 мс | 10551 мс |
| p95 | 14000 мс | 3 мс | 16000 мс |

**Основные различия:**

- Sanic быстрее Flask в 45 раз при 50 пользователях
- Sanic масштабируется линейно: 50u → 200u (82 → 327 RPS)
- Flask теряет производительность при росте нагрузки
